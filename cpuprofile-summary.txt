Root cause chain:
  1. File watcher detects file change
  2. Triggers Vcs.status() → spawns `git status --porcelain` + `git diff --numstat`
  3. For each dirty file, calls structuredPatch(file, before, after, { context: MAX_SAFE_INTEGER })
  4. structuredPatch runs Myers diff O(N·D) where N = file lines, D = edit distance
  5. With context: MAX_SAFE_INTEGER, the entire file is included in the patch output
  6. On large files (20k+ lines), this takes 100ms+ per file
  7. With 200+ dirty files × repeated watcher triggers = sustained 200%+ CPU
